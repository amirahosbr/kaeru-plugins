kaeru-plugins

Two plugins for shipping small website changes safely — one for the people who make them, one for the developer who reviews them.

Not a website or an app. It's a set of commands you install into Claude Code and then type — like /kaeru:start.

What's inside

Two plugins, two audiences

🐸
for non-technical staff

kaeru

Safely edit a website's text, images and colors — in your own language (Japanese or English) — and submit a PR for review. Git is hidden; it never merges.

15 commands · Playwright preview · PR-only
🐙
for the developer

octo

A GitHub + CI/CD toolkit: set up identity, bootstrap a repo, run the issue → PR → merge flow, watch CI, and scaffold the issue-driven templates into any repo.

Install

Add the marketplace, then pick a plugin

# 1) add the marketplace once
/plugin marketplace add amirahosbr/kaeru-plugins

# 2) install what you need
/plugin install kaeru@kaeru-plugins    # non-technical staff
/plugin install octo@kaeru-plugins     # the developer

/reload-plugins

The two are independent — install one or both. A repo's .claude/settings.json can auto-enable a plugin for whoever opens it.

The full flow

One change, end to end

How a small edit travels from a non-technical person to a shipped, reviewed change — 🐸 kaeru hands off to 🐙 octo at the PR.

  1. 🐸 staff/kaeru:startOpen a fresh, safe workspace
  2. 🐸 staff/kaeru:edit-text · edit-image · edit-colorMake the change, in plain language
  3. 🐸 staff/kaeru:previewSee it live and confirm
  4. 🐸 staff/kaeru:submitChecks pass → opens a PR (never merges)
  5. 🐙 octo/octo:fill-prFill the PR template from the actual diff
  6. 🐙 octo/octo:checksCI runs; git hooks guard lint / tests
  7. 👤 developerReviews the rendered preview — pixels, not diffs
  8. 🐙 octo/octo:mergeMerge once green and approved
  9. 🐙 octo/octo:releaseShip it
